# 配置说明

本页面详细介绍 LLBot Manager 涉及的各项配置，包括实例配置、LLBot 配置、PMHQ 配置和环境变量。

## 实例配置

实例配置在创建实例时通过 `POST /api/accounts` 请求体传入，也可通过 `PATCH /api/accounts/{id}` 更新。以下是完整的实例配置参数：

| 参数 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `name` | string | — | 实例名称，用于标识和展示 |
| `pmhq_mode` | string | `native` | PMHQ 运行模式：`native` 或 `external` |
| `llbot_entry` | string | 自动检测 | LLBot 入口路径 |
| `pmhq_host` | string | `127.0.0.1` | PMHQ 监听地址 |
| `pmhq_port` | integer | 自动分配 | PMHQ 通信端口 |
| `qq_path` | string | 自动检测 | QQ 客户端路径 |
| `pmhq_extra_args` | string | — | PMHQ 额外启动参数 |

### 创建实例示例

```bash
curl -X POST http://127.0.0.1:9090/api/accounts \
  -H "Content-Type: application/json" \
  -d '{
    "name": "production-bot",
    "pmhq_mode": "native",
    "pmhq_port": 13100,
    "qq_path": "C:/Program Files/Tencent/QQ/QQ.exe",
    "pmhq_extra_args": "--debug"
  }'
```

## LLBot 配置

每个实例拥有独立的 LLBot 配置文件，基于 `default_config.json` 模板生成。配置文件位于实例工作目录下：

```
llbot_manager_workspace/instances/{instance_id}/llbot/config/
```

### default_config.json 模板

以下是一个完整的 LLBot 默认配置模板，创建实例时 Manager 会基于此模板生成实例的配置文件，并将端口替换为实际分配的端口：

```json
{
  "webui": {
    "enabled": true,
    "host": "0.0.0.0",
    "port": 3080,
    "token": ""
  },
  "ob11": {
    "enabled": true,
    "host": "0.0.0.0",
    "port": 13000,
    "token": "",
    "enable_force": false,
    "debug": false,
    "heartbeat": 30000
  },
  "milky": {
    "enabled": true,
    "host": "0.0.0.0",
    "port": 3010,
    "token": ""
  },
  "satori": {
    "enabled": true,
    "host": "0.0.0.0",
    "port": 5600,
    "token": ""
  }
}
```

### 配置项说明

| 配置组 | 说明 |
| --- | --- |
| `webui` | LLBot Web 管理界面配置，包括是否启用、监听地址和端口 |
| `ob11` | OneBot 11 协议配置，用于与 PMHQ 通信 |
| `milky` | Milky 协议配置 |
| `satori` | Satori 协议配置 |

<callout type="info" title="端口自动替换">
创建实例时，Manager 会自动将配置模板中的端口替换为实际分配的端口。例如，实例 A 的 `ob11.port` 会被替换为 13000，`webui.port` 替换为 3080。
</callout>

### 自定义默认配置

如果你需要修改默认配置模板，可以通过环境变量 `LLBOT_DEFAULT_CONFIG` 指定自定义模板文件路径：

```bash
export LLBOT_DEFAULT_CONFIG=/path/to/my_default_config.json
llbot_manager --port 9090
```

```powershell
$env:LLBOT_DEFAULT_CONFIG = "C:\path\to\my_default_config.json"
.\llbot_manager.exe --port 9090
```

## PMHQ 配置

每个实例拥有独立的 PMHQ 配置文件 `pmhq_config.json`，位于实例工作目录下：

```
llbot_manager_workspace/instances/{instance_id}/pmhq_config.json
```

### 配置项说明

```json
{
  "headless": true,
  "quick_login_qq": "",
  "pmhq_port": 13000,
  "qq_path": "C:/Program Files/Tencent/QQ/QQ.exe",
  "extra_args": ""
}
```

| 配置项 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `headless` | boolean | `true` | 是否以无头模式运行 QQ（不显示 QQ 窗口） |
| `quick_login_qq` | string | — | 快速登录的 QQ 号码（留空则使用二维码登录） |
| `pmhq_port` | integer | 实例分配端口 | PMHQ 通信端口 |
| `qq_path` | string | 自动检测 | QQ 客户端路径 |
| `extra_args` | string | — | PMHQ 额外启动参数 |

<callout type="warning" title="手动修改配置">
直接修改 `pmhq_config.json` 文件后，需要重启实例才能使配置生效。建议通过 `PATCH /api/accounts/{id}` 接口更新配置，以确保配置与 state.json 同步。
</callout>

## 端口范围

Manager 默认的端口分配范围如下：

| 端口类型 | 默认起始端口 | 默认结束端口 | 说明 |
| --- | --- | --- | --- |
| PMHQ 端口 | 13000 | 13999 | PMHQ 通信端口 |
| WebUI 端口 | 3080 | 3179 | LLBot WebUI 端口 |
| Satori 端口 | 5600 | 5699 | Satori 协议端口 |
| Milky 端口 | 3010 | 3109 | Milky 协议端口 |

### 自定义端口

如果你需要手动指定端口，可以在创建实例时通过 `pmhq_port` 参数指定。Manager 会检查该端口是否可用，如果已被占用则返回错误。

## 环境变量

LLBot Manager 支持以下环境变量：

| 环境变量 | 默认值 | 说明 |
| --- | --- | --- |
| `LLBOT_DATA_DIR` | `llbot_manager_workspace` | Manager 工作目录路径 |
| `LLBOT_DEFAULT_CONFIG` | 内置模板 | LLBot 默认配置模板文件路径 |
| `LLBOT_HOST` | `0.0.0.0` | Manager HTTP 服务监听地址 |
| `LLBOT_PORT` | `9090` | Manager HTTP 服务监听端口 |
| `LLBOT_ENTRY` | 自动检测 | LLBot 入口路径 |
| `LLBOT_PMHQ_BINARY` | 自动检测 | PMHQ 可执行文件路径 |

<callout type="info" title="环境变量与命令行参数">
环境变量和命令行参数可以互换使用。命令行参数的优先级高于环境变量。如果同时设置了两者，命令行参数会覆盖环境变量的值。
</callout>

### 自定义工作目录示例

```bash
# 将工作目录设置到其他路径
export LLBOT_DATA_DIR=/data/llbot_manager
llbot_manager --port 9090
```

```powershell
$env:LLBOT_DATA_DIR = "D:\llbot_manager_data"
.\llbot_manager.exe --port 9090
```
